1. 参数处理

1.1 一个或多个参数的获取

  • 传单个参数：mybatis不会做处理，{}里面可以随便写
    #{参数名} 获得参数值
  • 多个参数:多个参数会被封装成一个map，key:param1…paramN; value传入的参数，而#{}就是从map中获取指定key的值。
    • 方式一：直接使用arg0,age1….有的参数是0,1,……：
    • 方式二：使用参数注解，命名参数，明确指定封装参数时的key
    • 方式三: 如果参数刚好是我们业务逻辑模型，我们可以直接传入pojo
    • 方式四： 如果参数不是我们的业务逻辑模型，我们可以传入map
    • 方式五：如果参数不是我们的业务逻辑模型，但是经常使用，我们可以写一个TO(transfer object)数据传输对象

特别注意：如果一个参数是Collection(List，Set)类型或是数组，也会把传入的list或者数组封装在map中。

key:Collection,如果是List还可以使用这个key(list)，数组(array)

#{list[0]}

1.2 #{}和${}的区别

  • #{}：表示一个占位符类似于preparedStatement中的?，可以自动进行java类型和jdbc类型的转化。#{}可以有效的防止SQL注入，可以接收简单数据类型或pojo属性。如果paramType传输的是单个简单类型,#{}括号中可以使value或其他名称
  • ${}：表示拼接SQL串，通过${}可以将paramType传入的内容拼接在SQL中且不进行jdbc类型转化，${}可以接收简单类型值或pojo属性值，如果parameterType传输单个简单类型值，${}括号中只能是value。
  • #{} ${}都可以获取pojo中的属性值，在大多数情况下我们使用#{}获取，在jdbc不支持占位符的地方使用${}
  • #{}:更丰富的用法：
    • 规定参数的些规则：javaType,jdbcType,mode(存储过程),numericScale,resultMap,typeHandler,jdbcTypeName,expression(未来准备支持的。。。。)

2 select查询

2.1 resultType封装类型

  • 封装一条记录为map,key是列名，value是对应的值，resultType=map
  • 封装多条记录map,键是这条记录的主键，值是封装的java bean,resultType=element

2.2 resultMap实现高级结果映射

resultMap可以自定义java bean的映射规则，resultMap有两个属性，type:返回的数据类型，id:唯一标识，方便引用。

resultMap可以使用的标签如下：

  • id:指定主键列的封装规则

    • column：指定哪一列
    • property：指定对应的javabean属性

  • result:定义普通列封装规则，不指定的列会自动封装。

  • collection

    • column：数据库中的联合的列，指定将哪一列的值传给select调用的方法
    • property:指定哪个属性是联合的对象，也就是属性名
    • javaType:指定联合对象的类型，也就是属性类型
    • ofType：指定集合元素类型
    • select：调用的查询方法id,表明当前属性是调用select方法返回的结果
    • fetchType=”lazy|eager”:设置是否延时加载

  • association：指定联合的java bean对象,定义单个对象的封装规则

    • column：数据库中的联合的列，指定将哪一列的值传给select调用的方法
    • property:指定哪个属性是联合的对象
    • javaType:指定联合对象的类型
    • select：调用的查询方法id,表明当前属性是调用select方法返回的结果

  • discriminator：鉴别器，mybatis可以使用discrimination判断某列的值，然后根据某列的值改变封装行为

    • column：指定判定的列名
    • javaType:列值对应的java类型

3. 动态SQL

动态SQL就是动态的构建SQL语句

  • if ：判断，主要就是用于拼接SQL语句，如果参数不为空则拼接上SQL语句
  • choose(when,otherwise)：分支选择

  • trim(where, set)：字符串截取

    • where 标签：只会去掉前面多出的and或or,所以使用where标签and要写在前面
    • trim标签：自定义字符串截取规则
      • prefix=”” ：前缀，给trim标签体中整个字符串加一个前缀
      • suffix=”” ：后缀
      • prefixOverrides=”” ：前缀覆盖，去掉字符串前面多余的字符
      • suffixOverrides=””：后缀覆盖
    • set标签：在更新时候设置set标签，会自动去掉后面的逗号，也可以用trim实现

  • foreach：遍历

    • collection：指定要遍历的集合(list|array)
    • open:以什么开始
    • separator：以什么分割
    • close：以什么结束
    • item：遍历的变量，用#{}取出
    • index：索引
