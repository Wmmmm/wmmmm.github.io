Mybatis基础—全局配置文件和基本使用

1.myBatis环境配置

首先要在GitHub上下载mybatis的jar包，需要用到的jar包只有一个，mybatis-3.4.5.jar。但是为了方便我们查看，用一起使用log4j.jar打印输出。

  1. 新建一个project
  2. 导入需要用到的jar包：，mybatis-3.4.5.jar、log4j.jar和mysql-connector-java-5.1.43-bin.jar
  3. 在project的根路径下创建一个sourceFolder，用于存放xml配置文件
  4. 编写mybatis的全局配置文件——mybatis-config.xml、SQL映射配置文件——UserMapper.xml
  5. 获取sqlSessionFactory对象，执行SQL

2.mybatis实现对数据库的连接

1、根据xml文件（全局配置文件），创建一个SQLSessionFactory的对象有数据源的一些映射信息

2、sql的映射文件，配置了每一个sql，以及sql的封装规则

3、将sql映射文件注册在全局配置文件中

4、写代码：

*     1）根据全局配置文件得到sqlSessionFactory </br>
*     2)使用sqlSessionFactory,获得sqlSession对象 </br>
*     3）使用sqlSession对象执行增删改查，使用sql的唯一标识告诉mybatis执行哪一个sql，sql都是保存在sql映射文件中的</br>
*     4）一个sqlSession就是代表和数据库的一次会话，用完关闭</br>

1、接口式编程：
原生： Dao –> DaoImpl

mybatis: Mapper –> xxMapper.xml

2、sqlSession代表和数据库的一次会话，用完关闭

3、sqlSession和Connection一样都是非线程安全的，所以说每次使用都应该获取新的对象

4、mapper接口虽然没有实现类，但是mybatis会为接口生成一个代理对象。
（将接口和xml绑定）

5、两个重要的配置文件，
一个是mybatis的全局配置文件，包含数据库连接池信息，事务管理器信息等等，系统运行环境信息
一个是mapper.xml数据库映射文件配置。包含了每一个sql语句的映射信息。

3. mybatis的全局配置文件

这些标签在编写的时候要按照顺序，不能乱写哦，可以不写某个标签但是不能打乱先后顺序。

  • properties:可以引入外部的数据源，有两个属性：
    • resource：可以引入类路径下的资源
    • url：可以引入网络或者磁盘路径下的资源
  • settings：包含了很多重要的设置项
    • setting：用来设置每一个设置项
      • name:设置项名
      • value：设置项取值
  • typeAliases:别名处理器，可以给java类型起一个别名，别名不区分大小写
    • typeAlias:为每一个java类起名
      • type:指定要起别名的java类的全类名；默认别名就是类名小写
      • alias:指定新的别名
    • package:批量起别名，可以为某个包下的所有类批量起别名
      • name:指定包名，为当前包和子包下的每一个类都起一个默认的别名

注意：批量起别名的方法在有相同类名的情况下会出错，这时可以使用注解的方式，在类上增加注解@Alias(“user”)为类添加新的别名。

mybatis已经为我们封装好了基本数据类型、他的包装类、Date、List等java常用的类起好了别名，可以直接使用。

  • typeHandlers:类型处理器，java数据类型和数据库数据类型之间的映射
  • plugins：插件配置，拦截sql
    • Executor
    • ParameterHandler
    • ResultSetHandler
    • StatementHandler
  • environments:环境配置标签，可以配置多种环境(开发环境和测试环境等)
    • environment:为每一个环境写配置信息，例如数据库,每一个environment标签必须要有transactionManager标签和dataSource标签。environment有一个id属性，唯一标识当前的环境，可以用于切换环境。
      • transactionManager:事务管理器，有一个属性type:事务管理器的类型，取值有：jdbc|manager|自定义。
      • dataSource：数据源，有一个属性type:取值可以为UNPOOLED|POOLED|JNDI|自定义(实现DataSourceFactory接口)
  • databaseIdProvider:允许根据数据库厂商执行不同的sql语句。
  • mappers:sql映射配置

  • mapper：每一个mapper注册一个SQL映射文件，有两个属性

    • resource:引用类路径下的SQL映射文件
    • url:引用网络路径下或磁盘的SQL映射文件
    • class:引用接口，如果要注册接口，两种实现方法:
      • 1.有sql映射文件，而且映射文件名必须和接口一致，并且存放在同一路径下
      • 2.没有SQL映射文件，所有的SQL都是利用注解写在接口上。

  • package:批量注册mapper，也是映射文件需要和接口同包名同文件名。

推荐：比较重要的、复杂的接口执行我们用SQL映射文件，简单的可以使用注解的方式。

4. mybatis的sql映射配置文件mapper

映射文件很重要啊，可以使用的标签有：

  • cache：配置给定命名空间的缓存。
  • cache-ref：从其他命名空间引用缓存配置。

  • resultMap：自定义结果集映射

  • sql：可重用的SQL块，也可以被其他语句引用。
    mapper标签有一个namespace：名称空间属性 ,指定为接口的全类名。

  • insert
  • update
  • delete
  • select
  • insert.update delete：三个比较类似，有一下几个属性：
    • id:唯一标识符指定为接口的方法名
    • parameterType：传入参数的类型
    • flushCache：设置为true，无论什么时候被调用都会使缓存清空，默认为false
    • timeout：设置驱动等待数据库返回请求结果
    • statementType：STATEMENT|PREPARED|CALLABLE中的一种
    • useGeneratedKeys：（仅对insert有用）通知MyBatis使用JDBC的getGeneratedKeys方法来取出由数据（如MySQL和SQL Server的数据库管理系统的自动递增字段）内部生成的主键。默认值为false。
    • keyProperty：（仅对insert有用）标记一个属性，MyBatis会通过getGeneratedKeys或insert语句的selectKey子元素设置其值。默认不设置。
  • select：映射查询语句。
    • id:唯一标识符指定为接口的方法名
    • parameterType：传入参数的类型
    • resultType：返回值类型
    • resultMap：命令引用外部的map
    • flushCache：设置为true，无论什么时候被调用都会使缓存清空，默认为false
    • useCache：设置为true，本条语句结果会被缓存，默认为true
    • timeout：设置驱动等待数据库返回请求结果
    • fetchSize：暗示驱动每次批量返回结果的行数，默认不设置，驱动自行处理
    • statementType：STATEMENT|PREPARED|CALLABLE
    • resultSetType：FORWARD_ONLY|SCROLL_SENSITIVE|SCROLL_INSERNSITIVE
  • cache：配置给定命名空间的缓存。
  • cache-ref：从其他命名空间引用缓存配置。
  • resultMap：最复杂，也是最有力量的元素，用于描述如何从数据库结果集中加载对象。
  • sql：可重用的SQL块，也可以被其他语句引用。
